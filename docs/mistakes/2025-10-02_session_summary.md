## 2025년 10월 2일 실수 요약

### 1. `AppHeader.vue` 리팩토링 중 저장 기능 회귀 버그 발생
- **문제:** 반응형 헤더 구현 과정에서 기존 `AppHeader.vue`에 있던 저장 버튼 및 관련 로직(페이지 제목 동적 표시, `usePageStore`, `useRoute` 등)을 실수로 삭제하여 저장 기능이 동작하지 않음.
- **원인:** 새로운 기능(반응형 헤더) 구현에 집중하면서 기존 컴포넌트의 핵심 로직을 충분히 고려하지 않고 덮어쓰기 방식으로 수정.
- **해결:** 삭제된 로직을 `AppHeader.vue`에 다시 병합하여 복원.
- **핵심 배움:** 대규모 컴포넌트 리팩토링 시, 기존의 중요한 기능 로직이 누락되지 않도록 주의 깊게 병합하거나, 리팩토링 전 해당 기능에 대한 테스트 코드를 확보하는 것이 중요.

### 2. `AppSidebar.vue` 로고 적용 시 사용자 아바타 삭제
- **문제:** 사이드바에 로고를 적용하는 과정에서 사용자 아바타를 로고로 대체하여, 로그인 후 아바타가 보이지 않고 로고만 표시됨.
- **원인:** 사용자 요청을 잘못 해석하여 아바타를 로고로 대체하는 방식으로 구현.
- **해결:** `AppSidebar.vue`를 원래 상태로 되돌려 아바타가 다시 표시되도록 수정.
- **핵심 배움:** UI 요소 배치 변경 시, 기존 요소의 역할과 사용자 경험을 충분히 고려하고, 모호한 요청은 재확인하여 의도를 명확히 파악해야 함.

### 3. `docs:tree` 스크립트 구현 과정에서의 반복적인 환경/문법 오류
- **문제:** 프로젝트 파일 트리 자동 생성 스크립트 구현 과정에서 윈도우 환경, `npm` 스크립트, Node.js 모듈 시스템 관련 오류가 반복적으로 발생.
- **원인:**
    - **윈도우 `tree` 명령어의 `-I` 옵션 미지원:** 리눅스 `tree` 명령어의 기능을 윈도우 기본 `tree`가 지원할 것이라는 잘못된 가정.
    - **`treenode` 패키지 실행 문제:** `npm exec`가 `treenode` 패키지의 실행 파일을 찾지 못함. (패키지 자체의 문제 또는 `npm`과의 호환성 문제)
    - **`npm` 스크립트 내 셸 인용(quoting) 문제:** `npx tree -I node_modules|dist|...`와 같이 `|` 문자가 포함된 문자열이 셸에 의해 잘못 해석됨.
    - **Node.js ES 모듈 환경에서 `require()` 사용:** `package.json`의 `"type": "module"` 설정으로 인해 `.js` 파일이 ES 모듈로 처리되는데, `require()` 구문을 사용하여 `ReferenceError` 발생.
- **해결:**
    - `tree-node-cli` 대신 `tree-cli` 패키지 사용.
    - `package.json` 스크립트 내에서 제외 패턴 문자열을 `\"...\"`와 같이 정확하게 인용 처리.
    - Node.js 스크립트(`scripts/generateTree.js`)를 `require()` 대신 `import` 구문을 사용하는 ES 모듈 문법으로 변환.
- **핵심 배움:**
    - 개발 환경(OS, Node.js 모듈 시스템)에 대한 깊은 이해와 `npm` 스크립트 작성 시 셸의 인용 규칙에 대한 정확한 적용이 필수적.
    - 크로스 플랫폼 호환성을 고려한 도구 선택 및 사용법 숙지.
    - 오류 발생 시, 근본적인 환경 설정을 재확인하는 습관.
